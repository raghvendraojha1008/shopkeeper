fix: resolve Android add/edit entry failures caused by memory pressure

PROBLEM
-------
Users reported inability to add/edit entries on Android devices, with generic 
"Save failed" errors providing no diagnostic information. Analysis of logcat 
revealed severe GPU memory pressure (MALI BAD_ALLOC errors) causing IndexedDB 
write operations to block for >15 seconds, exceeding the original 8-second timeout.

ROOT CAUSE
----------
1. Write timeout (8s) too short for Android GC pauses under memory pressure
2. Inadequate error reporting masked real causes (permission, offline, timeout)
3. No diagnostic visibility into failures (no logging, no debug UI)
4. Memory pressure on low-RAM devices causes IndexedDB operations to block

CHANGES
-------

1. Increased Write Timeout (CRITICAL FIX)
   - File: src/services/api.ts
   - Changed: WRITE_TIMEOUT_MS from 8_000ms to 15_000ms
   - Reason: Allow IndexedDB writes to complete during extended Android GC pauses
   - Impact: Fixes saves on low-memory devices without affecting timeout sentinel

2. Enhanced Error Messages
   - File: src/components/modals/ManualEntryModal.tsx
   - Added: Specific error categorization (timeout vs permission vs offline)
   - Reason: Users can now understand why saves fail and take appropriate action
   - Impact: Better UX with actionable error messages

3. Diagnostic Logging System
   - File: src/utils/diagnosticLogger.ts (NEW)
   - Added: Comprehensive error capture and formatting
   - Features: Up to 100 entries, exportable to clipboard/console
   - Impact: Enables post-mortem debugging without USB access

4. Debug Panel for Real-time Diagnostics
   - File: src/components/common/DiagnosticsPanel.tsx (NEW)
   - Added: Keyboard shortcut (Ctrl+Shift+D) to view error logs
   - Features: Copy to clipboard, dump to console, live error viewing
   - Impact: Users can self-diagnose issues without developer tools

5. Documentation
   - Added: ANDROID_ADD_EDIT_TROUBLESHOOTING.md (comprehensive guide)
   - Added: ANDROID_FIX_SUMMARY.md (technical analysis)
   - Added: QUICK_FIX_REFERENCE.md (quick reference card)
   - Impact: Clear path for users and developers to debug issues
