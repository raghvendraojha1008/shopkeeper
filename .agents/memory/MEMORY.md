- [Toast invalid-type crash](toast-invalid-type-crash.md) — `'warning'` passed to showToast crashes Toast render; only `'success'|'error'|'info'` are valid types.
- [Anonymous block let scope bug](anonymous-block-let-scope.md) — `let` vars declared inside `{ }` anonymous block in ManualEntryModal caused ReferenceError outside block; fix: declare before the block.
- [PDF generation strategy](pdf-generation-strategy.md) — jsPDF is stubbed (vite alias), native PdfGenerator plugin crashes GPU on Android; use purePdf (src/utils/purePdf.ts) as the working fallback.
- [PDF Tm operator syntax](pdf-tm-operator.md) — PDF `Tm` operator needs 6 args (`1 0 0 1 x y Tm`), not 2. Passing 2 args silently drops ALL text; path ops (borders) still draw.
- [SyncQueue pending-forever fix](sync-queue-retry.md) — SyncQueue only retries on offline→online event; if device is online-but-slow, call processQueue(uid) immediately after addToQueue in WRITE_TIMEOUT path.
- [Party cascade party_id](party-id-cascade.md) — partySync queries both party_id AND party_name for all collections; ManualEntryModal stamps party_id from safeParties cache on every new ledger/txn entry.
- [Firebase cache — persistentLocalCache](firebase-cache-decision.md) — NEVER use memoryLocalCache; it requires live network per write → stuck pending. Use persistentLocalCache+forceOwnership:true.
- [Non-blocking save pattern](non-blocking-save.md) — ManualEntryModal uses ApiService.prepareBatch (synchronous, no await); IDs generated locally; modal closes instantly; commit fires in background; cascade inside .then().
- [Party filter must use party_id](party-filter-party-id.md) — PartyDetailView and PartyStatementView must filter by `party_id === party.id || party_name === party.name`; name-only filter breaks immediately after rename while cache still carries old party_name.
- [PartiesView accounting must use party_id keys](parties-view-accounting-keys.md) — ledgerByParty/transactionsByParty must be keyed by party_id (not party_name) so balance is stable during rename; legacy records use `name:${party_name}` prefix key as fallback.
- [partySync cache patch pattern](party-sync-cache-patch.md) — after each flushUpdates() in partySync.ts, call queryClient.setQueryData to patch ledger/transactions caches in-memory immediately; invalidateQueries is a secondary background reconciliation, not the primary update.
- [Duplicate records from SyncQueue+SDK race](duplicate-records-sync-race.md) — ManualEntryModal background commit catch must NOT queue to SyncQueueService when online; with memoryLocalCache the Firestore SDK retries internally, so both paths write → duplicates. Only queue when `!navigator.onLine`.
- [PDF text fallback encoding](pdf-text-fallback-encoding.md) — Capacitor Filesystem `encoding: 'utf8' as any` fails on some Android WebView versions with non-ASCII content; fix: TextEncoder→base64 then write without encoding param.
- [Inventory item_id stamping](inventory-item-id-stamping.md) — ManualEntryModal must stamp `_item_id` on lineItem in handleItemChange when autocomplete finds a match; submit-time must match by ID first then name. Prevents spurious inventory ADD for existing items when TanStack cache is stale.
- [POS offline pattern](pos-offline-pattern.md) — POSBillingView.persist() uses prepareBatch unified path (no isOffline branching); party/ledger/inventory ops all batched; commit is fire-and-forget; SyncQueue only on !navigator.onLine in catch.
- [Inventory optimistic cache update](inventory-optimistic-update.md) — After capturedCommit() resolves, call setInventoryData() with the batch changes BEFORE refetchInventory() so the next modal open immediately sees correct stock without waiting for async getDocs.
