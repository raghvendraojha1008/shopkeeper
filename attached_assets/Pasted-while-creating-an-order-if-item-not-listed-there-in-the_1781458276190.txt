Root causes found & fixed
Bug 1 — Cache never refreshed (the main culprit)
useInventory is a one-shot fetch with a long stale time — not a real-time listener. After batchSave wrote new/updated items to Firestore, React Query's cache stayed frozen. The inventory list in the form's autocomplete wouldn't update, and the inventory screen wouldn't show new items until you navigated away and came back.

Fix: refetchInventory() is called immediately after any successful batchSave that touched inventory, forcing the cache to sync with Firestore right away.

Bug 2 — Silent operation, no feedback
The user had no way to know if anything actually happened after saving an order.

Fix: Two new toasts now appear after save:

"2 items added to inventory" — when new items were auto-created
"3 inventory items updated" — when existing items had their rates/unit/GST synced
Bug 3 — "Add to stock" showed on all items, including existing ones
The checkbox appeared even when the item name was already in your inventory list, which was confusing and implied it would do something it wouldn't (the update path handles existing items differently).
