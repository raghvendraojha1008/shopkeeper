You are working on a production-grade React + Capacitor + Firebase (Firestore + Auth) application. Your task is to design and implement a **complete, scalable Subscription Management System** from scratch, integrated cleanly into the existing architecture.

Do NOT implement a partial or UI-only solution. Build a **full system** including data model, access control, lifecycle handling, and future payment readiness.

---

# 🎯 OBJECTIVE

Create a robust subscription system that:

* Controls feature access across the app
* Supports plans (free, pro, etc.)
* Handles lifecycle (active, trial, expired)
* Is payment-ready (but initially free ₹0 rollout)
* Is secure, consistent, and scalable

---

# 🧠 SYSTEM DESIGN (MANDATORY)

Before coding, ensure the system includes:

## 1. Single Source of Truth (Firestore)

Create a structured subscription document:

```
users/{userId}/subscription/current
```

Schema:

```
plan: "free" | "pro" | "enterprise"
status: "active" | "trial" | "expired"
startDate: timestamp
endDate: timestamp
trialUsed: boolean
autoRenew: boolean
paymentId: string | null
createdAt: timestamp
updatedAt: timestamp
```

Rules:

* Only ONE active subscription per user
* Always read from Firestore (no hardcoding)

---

## 2. Subscription Context (Global State)

Create a `SubscriptionContext` similar to AuthContext:

* Fetch subscription on login
* Cache in state
* Provide globally:

  * subscription object
  * loading state
  * refresh function

Ensure:

* No redundant fetches
* Handles user switch
* Handles offline fallback safely

---

## 3. Feature Access Control Layer (CRITICAL)

Create a centralized utility:

```
hasAccess(subscription, featureKey)
```

Define feature mapping:

```
const PLAN_FEATURES = {
  free: ["basic"],
  pro: ["basic", "analytics", "reports", "advanced_features"],
  enterprise: ["all"]
}
```

Rules:

* NEVER hardcode access in UI
* ALL feature gating must use this function

---

## 4. Feature Gating Integration

Apply access control across app:

Examples:

* Analytics dashboard
* Reports
* Advanced views
* Future premium features

Behavior:

* If access denied → show upgrade prompt
* Do NOT crash or hide silently
