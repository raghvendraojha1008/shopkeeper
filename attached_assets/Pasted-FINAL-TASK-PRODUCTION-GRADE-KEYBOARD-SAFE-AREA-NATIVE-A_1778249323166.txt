FINAL TASK — PRODUCTION-GRADE KEYBOARD, SAFE-AREA & NATIVE ANDROID BEHAVIOR SYSTEM

SYSTEM CONTEXT

This is a production-stage Capacitor + React Android application with:

* bottom navigation
* forms
* overlays
* modals
* sheets
* offline workflows
* large lists
* POS flows

The goal is NOT simply to “fix keyboard overlap”.

The goal is to create a native-quality Android interaction system where:

* keyboard behavior
* viewport resizing
* focus transitions
* safe-area handling
* touch interactions
  all feel physically correct and polished.

The app should feel indistinguishable from a well-built native Android app.

---

PRIMARY OBJECTIVE

Implement a centralized keyboard and native-behavior architecture that ensures:

* predictable layout behavior
* safe-area correctness
* stable viewport resizing
* proper focus handling
* interruption-safe interactions

across all screens and device sizes.

---

CORE PRINCIPLE

Keyboard appearance/disappearance must NEVER:

* cover inputs
* break layout
* trap scrolling
* overlap buttons
* cause viewport jumps
* create hidden actions

The UI should adapt smoothly and predictably.

Use:

* viewport listeners
* focus tracking

Avoid:

* deterministic keyboard handling
