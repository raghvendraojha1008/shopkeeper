You are working on the Shopkeeper system (React + Capacitor + Firebase + Firestore).
The subscription system and Admin Panel are already implemented and aligned with Firestore contracts.

Your task is to implement a **complete, production-grade Payment + Verification + Renewal System**, including backend validation and webhook handling.

---

# 🧠 SYSTEM ARCHITECTURE (UNDERSTAND FIRST)

There are 4 layers:

1. **Frontend App**

   * Initiates payment
   * Shows UI
   * NEVER final authority

2. **Backend (Cloud Functions / Server)**

   * Verifies payment
   * Writes subscription (ONLY trusted layer)

3. **Razorpay Gateway**

   * Handles payment
   * Sends webhook

4. **Firestore (Source of Truth)**

---

# 🎯 OBJECTIVE

Build a **secure, idempotent, backend-driven payment system** that:

* Prevents fraud
* Handles retries safely
* Supports renewals
* Works with webhook confirmation
* Fully syncs with Admin Panel

---

# 🚨 NON-NEGOTIABLE RULES

❌ Frontend must NOT directly activate subscription
❌ Never trust frontend payment success blindly
❌ Never write subscription without verification

✅ Backend (Cloud Function) is the ONLY authority
✅ All writes must be idempotent
✅ All events must be logged

---

## 1. create-order function

1. Verify signature using Razorpay secret
2. Check order + payment match

4. Apply renewal logic
5. Write subscription
