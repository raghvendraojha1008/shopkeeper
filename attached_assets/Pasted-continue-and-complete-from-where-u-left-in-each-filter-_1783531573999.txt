The root cause was a race condition in the touch event sequence:

User taps arrow → onTouchStart fires → list opens (setShowFullList(true))
React re-renders → the dark backdrop appears on screen
The same touch's synthesized click event fires ~50ms later — and now lands on the backdrop → setShowFullList(false) → list closes immediately ← the open-close cycle
Additionally, both onMouseDown + onTouchStart were registered, which fires the handler twice on Android.

The fix — three changes working together:

What	Change	Why
Arrow button	onPointerDown + preventDefault() (replaces onMouseDown + onTouchStart)	Fires before any focus event; preventDefault() suppresses the synthesized click that would hit the backdrop
Backdrop	onPointerDown + 400 ms guard (listOpenedAt.current)	Even if a touch reaches the backdrop, it's ignored within 400 ms of the sheet opening
Sheet content	onPointerDown={e => e.stopPropagation()}	Taps inside the sheet don't bubble to the backdrop
Inner list search	Removed autoFocus	List opens without forcing the keyboard — user taps the search field inside the sheet if they want to type-filter
showSuggestions	Added && !showFullList guard	Inline dropdown and full sheet can never both be visible at once
