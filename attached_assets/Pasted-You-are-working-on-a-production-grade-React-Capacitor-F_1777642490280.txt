# 🔧 ISSUE 1 — Keyboard causes blank space (CRITICAL UI BUG)

### Problem:

When the mobile keyboard opens, a large blank area appears and hides content.

### Root Cause:

* Root container uses fixed height (`h-screen`)
* Layout uses `overflow-hidden`
* Capacitor + mobile viewport resizing is not handled

### Fix:

* Replace all `h-screen` with `min-h-screen`
* Replace `overflow-hidden` on main container with `overflow-y-auto` where needed
* Ensure `<main>` scrolls, not the root container
* Avoid fixed viewport locking anywhere

### Validate:

* Test on Android device
* Input fields should remain visible when keyboard opens
* No empty space at bottom
