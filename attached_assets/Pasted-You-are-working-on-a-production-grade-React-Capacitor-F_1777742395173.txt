# 🚨 CRITICAL TASKS (DO FIRST)

---

## 1. 🔐 SERVICE-LEVEL ACCESS ENFORCEMENT (MANDATORY)

### Problem:

Users can bypass UI gating and still trigger logic

### Task:

Enforce access inside:

* ApiService (all create/update/delete operations)
* Data processing functions
* Any premium feature execution

### Pattern:

```ts
if (!hasAccess(subscription, "feature_key")) {
  throw new Error("Subscription required");
}
```

### Requirement:

* No premium logic should run without validation
* UI gating alone is NOT sufficient

---

## 2. ⚠️ NAVIGATION GUARD (PREVENT ENTRY)

### Problem:

User can navigate to restricted tabs before FeatureGate blocks UI

### Task:

Before setting activeTab:

```ts
if (!hasAccess(subscription, "feature")) {
  showUpgradePrompt();
  return;
}
```

### Apply to:

* bulk import
* analytics
* pos billing
* stock valuation
* timeline
* snapshots

---

## 3. ⚠️ BACKGROUND SERVICES MUST RESPECT SUBSCRIPTION

### Problem:

These run for all users:

* AutoBackupService
* DailySnapshotService
* AutoReminderService

### Task:

Wrap execution:

```ts
if (hasAccess(subscription, "daily_snapshot")) {
  runService();
}
```

### Requirement:

* Free users must NOT receive premium automation

---

## 4. ⚠️ FIX MOBILE KEYBOARD LAYOUT BUG

### Problem:

Root container uses fixed height → causes blank space when keyboard opens

### Fix:

* Replace `h-screen` → `min-h-screen`
* Remove `overflow-hidden` from root container
* Ensure main uses `overflow-y-auto`

---

### Rule:

## 6. ⚠️ SUBSCRIPTION IN LOGIC LAYER

### Problem:

## 7. ⚠️ SUBSCRIPTION LOADING STATE

### Problem:

UI renders before subscription is ready

### Task:

### Problem:

Ensure consistency across app
