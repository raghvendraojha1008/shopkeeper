# 🔥 MODULE 1 — ULTRA-FAST POS OPTIMIZATION

## Purpose

Reduce cognitive + interaction latency

## Current flow

Search → tap → add → edit → save

## Target flow

```text
Scan/Search → Auto-add → Next item (no friction)
```

---

## Requirements

### 1. Auto-focus loop

* After adding item → focus returns to search input
* No manual taps required

---

### 2. Optimistic cart updates

* Add instantly to UI
* Sync happens in background

---

### 3. Smart quantity logic

* Tap same item → increment quantity
* Long press → open edit

---

### 4. Preloaded inventory cache

* Load once → use in-memory
* No API call per search

---

## Edge cases

* Out of stock
* Network delay
* Duplicate item entries

---

## Output

```text
User can complete bill in < 5 seconds
```

---

# 🔥 MODULE 2 — OFFLINE-FIRST ARCHITECTURE

## Purpose

App must work without internet

---

## Current state

* Write queue exists ✅
* Read still depends on Firebase ❌

---

## Build this system

### 1. Local database layer

Use:

* IndexedDB (via Dexie or idb)

Store:

```text
inventory
ledger_entries
transactions
parties
```

---

### 2. Read flow

```text
UI → Local DB → (instant)
        ↓
     Firebase (sync)
```

---

### 3. Write flow

```text
Action → Save locally → mark "pending"
           ↓
        Sync queue → Firebase
```

---

### 4. Sync engine

Must handle:

* Retry on failure
* Conflict resolution
* Timestamp-based overwrite

---

## Edge cases

* App closed mid-sync
* Duplicate writes
* Partial sync

---

## Output

```text
App fully usable offline (read + write)
```

---
