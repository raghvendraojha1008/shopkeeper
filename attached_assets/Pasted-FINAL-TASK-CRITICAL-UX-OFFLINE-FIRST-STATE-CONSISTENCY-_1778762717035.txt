ISSUE 3 — OFFLINE-FIRST IS NOT FULLY TRUE OFFLINE-FIRST

CRITICAL.

Current issue:

They only appear after reconnect/sync.

This breaks true offline-first expectations.

Required behavior:
When user creates:

* ledger entry
* POS bill
* inventory update
* transaction

while offline:

The entry MUST:

1. appear instantly in UI
2. survive restart/process death
3. remain visible while still offline
4. later reconcile with server after reconnect

This is CRITICAL.

Audit entire optimistic persistence flow:

* React Query cache
* IndexedDB persistence timing
* queue persistence
* hydration timing
* optimistic temp entry handling

Requirements:

* optimistic entries persist offline immediately
* entries survive app restart before reconnect
* no disappearing temp entries
* no “online-only visibility”
* no stale hydration gaps

Likely issue:
Optimistic cache exists only in memory before persistence flush.

Fix properly.
Do NOT patch superficially.

Need true durable offline-first behavior.
