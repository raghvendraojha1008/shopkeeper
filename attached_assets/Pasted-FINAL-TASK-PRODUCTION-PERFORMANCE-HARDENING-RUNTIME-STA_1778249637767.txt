FINAL TASK — PRODUCTION PERFORMANCE HARDENING & RUNTIME STABILITY AUDIT

SYSTEM CONTEXT

This is a production-stage Capacitor + React Android application with:

* offline-first architecture
* React Query persistence
* optimistic updates
* sync queue
* lifecycle recovery
* overlays/modals
* forms
* analytics
* large datasets
* navigation stacks

The goal is NOT superficial optimization.

The goal is to harden the runtime system so the app remains:

* responsive
* memory-safe
* battery-efficient
* stable under long-term usage
* performant on low-end Android devices

The app should remain smooth and predictable under real-world conditions.

---

PRIMARY OBJECTIVE

Perform a deep runtime-performance and stability audit across:

* rendering
* memory
* event listeners
* persistence
* lists
* async operations
* sync flows
* navigation
* lifecycle recovery
* overlays

Then implement targeted hardening improvements WITHOUT breaking existing architecture.

---

CORE PRINCIPLE

The app must:

* remain responsive during long sessions
* avoid memory growth over time
* avoid unnecessary rerenders
* avoid excessive background work
* scale safely with larger datasets

Performance work must prioritize:

1. Stability
2. Predictability
3. Runtime efficiency
4. User-perceived smoothness

NOT premature micro-optimizations.
