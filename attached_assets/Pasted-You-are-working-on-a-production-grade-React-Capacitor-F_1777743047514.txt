You are working on a production-grade React + Capacitor + Firebase (Firestore + Auth) application that already has a subscription system implemented.

Your task is to **prepare the existing system for a future Super Admin Panel**, without actually building the admin panel now.

This means:
👉 You must **refactor and expose clean connection points (interfaces, data structures, APIs)** so that another agent can later plug in a Super Admin system **without breaking anything or requiring refactoring**.

---

# 🎯 OBJECTIVE

Make the subscription system:

* Fully controllable from an external admin panel (future)
* Decoupled from hardcoded logic
* Data-driven (Firestore-based, not code-based)
* Safe, consistent, and extensible

---

# 🧠 CORE PRINCIPLE

Separate system into 3 layers:

1. **Execution Layer (already exists)**

   * FeatureGate
   * hasAccess()
   * SubscriptionContext

2. **Control Layer (to be added later)**

   * Admin panel (NOT in this task)

3. **Data Layer (YOU MUST PREPARE NOW)**

   * Plans
   * Global config
   * Subscription records

👉 You are building the **data + interface contract** for future admin control.

Ensure structure is strict and future-proof:

```id="yk6qrg"

```

