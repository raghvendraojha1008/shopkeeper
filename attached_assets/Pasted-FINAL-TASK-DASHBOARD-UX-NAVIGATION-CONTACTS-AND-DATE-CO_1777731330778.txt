MODULE 6 — DATE CONSISTENCY FIX (CRITICAL)

Problem:
Dates in Ledger/Transaction lists show one day earlier than selected date

Root cause:
Date strings parsed as UTC instead of local time

Fix:

1. Replace ALL date parsing logic:

DO NOT use:

* new Date("YYYY-MM-DD")
* new Date(dateString)

2. Use SAFE LOCAL parsing:

function parseRecordDate(raw) {
if (!raw) return new Date(0);

if (raw?.toDate) return raw.toDate();

const s = String(raw);

if (s.includes('T')) return new Date(s);

const [y, m, d] = s.split('-').map(Number);
return new Date(y, m - 1, d); // LOCAL date
}

3. Ensure:

* LedgerView uses this
* TransactionsView uses this
* Sorting also uses parsed date

4. Verify:

* Selected date == displayed date
* No timezone shift anywhere
